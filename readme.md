# stories_v2 — Create a Story (API Workflow)

All routes are prefixed `/api/v2/`. JSON in, JSON out. No auth (DRF
`AllowAny`). v1 endpoints under `/api/stories/...` are unchanged.

The story-creation flow is split into discrete steps so the frontend can
let the user review + edit between each one.

---

## 0. Optional preflight: confirm wiring

### `GET /api/v2/health/`

Reports which AI providers are configured and which Mongo collections
exist. Use this once after deploying to confirm `GROQ_API_KEY` +
`TOGETHER_API_KEY` are picked up.

**Body:** none

**Response 200:**
```json
{
  "ok": true,
  "version": "v2",
  "atlas_detected": false,
  "collections_present": ["profiles_v2", "stories_v2", "character_bibles_v2", "world_bibles_v2", "beat_sheets_v2", "scene_drafts_v2", "run_events_v2"],
  "providers_configured": ["groq", "together"]
}
```

## 1. (Optional) Create a reusable author profile

### `POST /api/v2/profiles/generate/`

Builds a structured author voice (lexical fingerprint, emotional defaults,
banned/preferred phrases, few-shot samples) from a bio + optional writing
samples. Reusable across many stories — pass the returned `profile_id`
into a Quick or Deep survey to bake your voice into the prose.

**Body:**
```json
{
  "name": "M. Holt",
  "bio_context": "Frontier doctor turned novelist; uses short sentences, dry humour.",
  "background": "Grew up in mining towns.",
  "personality": "Stubborn, observational, suspicious of metaphor.",
  "communication_style": "Direct. Quiet. Withholds praise.",
  "interests_and_values": "Self-reliance, herbal medicine, blizzards.",
  "quirks": "Never names emotions outright.",
  "additional_context": "",
  "writing_samples": [
    "Sample passage one in the author's voice...",
    "Another passage..."
  ],
  "expertise_tags": ["literary", "frontier", "medical"]
}
```

**Response 201:**
```json
{
  "ok": true,
  "profile_id": "5a7b...",
  "name": "M. Holt",
  "preview": {
    "lexical_fingerprint": { "avg_sentence_words": 9.2, "contraction_rate": 0.41, "formality": 4, "style_register": "casual" },
    "emotional_defaults": { "default_valence": 0.0, "humor_type": "dry", "interiority_density": "medium" },
    "preferred_phrases": ["I'd say", "look"],
    "banned_phrases": ["delve", "unwavering"],
    "few_shot_count": 5
  }
}
```

## 2. Create the story envelope + bibles

Pick **one** of the three intake paths. All three persist a story envelope
(`stories_v2`) + a world bible (`world_bibles_v2`) + a tier-budgeted cast
(`character_bibles_v2`), then return a `story_id`. **Generation does NOT
start yet** — call `/start/` after the user reviews the bibles.

### 2a. Quick wizard (5-field, ~5 min)

`POST /api/v2/stories/quick/`

**Body:**
```json
{
  "title": "The Long Winter",
  "premise": "A frontier doctor faces an impossible blizzard with one nurse and no antibiotics.",
  "num_chapters": 20,
  "initial_chapters": 3,
  "genres": ["literary", "frontier"],
  "tone": ["gritty", "hopeful"],
  "characters": [
    { "name": "Mara Holt", "role": "protagonist", "trait": "stubborn doctor", "sample_line": "I'd say you've got about an hour." }
  ],
  "pov": "third_limited",
  "tense": "past",
  "target_chapter_words": 2200,
  "profile_id": "5a7b..."
}
```

**Response 202:**
```json
{
  "ok": true,
  "story_id": "9f7e...",
  "status": "pending",
  "bibles": {
    "world_bible_id": "9f7e...",
    "cast_size": 11,
    "tier_counts": { "main": 2, "recurring": 4, "side": 5 }
  },
  "note": "Bibles persisted. POST /api/v2/stories/<id>/start/ to begin writing."
}
```

### 2b. Deep wizard (full bibles upfront)

`POST /api/v2/stories/deep/`

Same shape, but `quick` is nested and you supply full character + world
detail. Anything you don't fill gets generated by Character Forge /
World Builder respecting your overrides.

**Body:**
```json
{
  "quick": { /* same as Quick body above */ },
  "characters": [
    {
      "name": "Mara Holt",
      "tier": "main",
      "role": "protagonist",
      "background": "Trained in Boston, came home after her father's death.",
      "motivations": ["keep everyone alive through winter"],
      "fears": ["losing another patient she could have saved"],
      "arc": "Learns to ask for help.",
      "speech_traits": "short sentences, dry humour, never names emotions",
      "preferred_phrases": ["I'd say", "look"],
      "banned_phrases": ["delve", "unwavering"],
      "sample_lines": ["I'd say you've got about an hour.", "Look. Sit down. Drink that."]
    }
  ],
  "world": {
    "setting": "alpine frontier town, late 1890s",
    "time_period": "1894-1895",
    "technology_level": "early industrial, no antibiotics",
    "banned_anachronisms": ["smartphone", "automobile"],
    "must_have_vibes": ["coal smoke", "frozen pump handles"]
  },
  "arc_preferences": {
    "must_include_tropes": ["the long winter", "deathbed promise"],
    "must_avoid_tropes": ["love triangle"],
    "cliffhanger_intensity": "medium",
    "pacing_speed": "slow_burn",
    "romance_temperature": "subtext",
    "action_density": "light"
  },
  "style_anchors": {
    "reference_authors": ["Annie Proulx"],
    "pasted_sample_passages": []
  }
}
```

**Response 202:** same shape as Quick.

### 2c. Paste-anything parser (pre-fill Deep wizard)

`POST /api/v2/stories/parse/`

Takes free text (handwritten notes, draft chapter, character bible)
and returns a structured `ParsedSurveyDraft` the user reviews + edits
before submitting to `/deep/`.

**Body:**
```json
{
  "raw_text": "<paste anything: notes, world bible, sample chapter...>",
  "hint": "this is a world bible for a cultivation web novel"
}
```

**Response 200:**
```json
{
  "ok": true,
  "draft": {
    "quick": { "title": "...", "premise": "...", "genres": ["cultivation"] },
    "characters": [ /* DeepCharacter shape */ ],
    "world": { /* DeepWorld shape */ },
    "notes": ["Inferred POV from sample passage.", "Magic system left blank — please fill."]
  }
}
```

## 3. (Optional) Review + edit the bibles before generation

The user can fix anything Character Forge / World Builder got wrong
before kicking off the engine.

### `GET /api/v2/bibles/<story_id>/characters/`

Lists all cast members with full bibles + voice fingerprints + mood
history. Pass `?tier=main` to filter.

### `GET /api/v2/bibles/<story_id>/character/<character_id>/`

One character's bible.

### `PATCH /api/v2/bibles/<story_id>/character/<character_id>/patch/`

**Body:** any subset of CharacterBibleV2 fields (name, role, motivations,
voice_fingerprint, etc.). Validates the merged document; rejects with
400 on a bad shape.

### `GET /api/v2/bibles/<story_id>/world/`

The world bible.

### `PATCH /api/v2/bibles/<story_id>/world/patch/`

**Body:** any subset of WorldBibleV2 fields.

## 4. Start writing

### `POST /api/v2/stories/<story_id>/start/`

Kicks off the generation engine in a daemon thread. Returns immediately.

Pipeline: Architect (macro arc) → Chapter Planner (Kishōtenketsu beats)
→ per scene { draft → 5 critics in parallel → editor → humanise →
continuity refresh → commit }. Writes `initial_chapters` chapters by
default; pass `batch_size` to override.

**Body (optional):**
```json
{ "batch_size": 3 }
```

**Response 202:**
```json
{
  "ok": true,
  "story_id": "9f7e...",
  "status": "queued",
  "note": "Generation started in background. Poll GET /api/v2/stories/<id>/ or subscribe to /stream/ for events."
}
```

**409** if the story is already `running` / `writing` / `architecting`
/ `completed`.

## 5. Watch it write (live)

### `GET /api/v2/stories/<story_id>/stream/?after_seq=N`

Server-Sent Events stream. `?after_seq=N` replays any missed events
before the live tail begins. Heartbeats every 15s keep proxies alive.

**Body:** none. **Response:** `text/event-stream` with frames like:

```
event: scene.started
id: 12
data: {"chapter_idx": 0, "scene_idx": 1, "pov": "Mara Holt", "target_words": 600}

event: scene.committed
id: 17
data: {"chapter_idx": 0, "scene_idx": 1, "word_count": 612, "summary": "Mara returns home...", "humanisation": {...}}
```

**Event types emitted (in order):**
`story.queued` → `arc.planned` → `chapter.planning` → `chapter.planned`
→ (per scene) `scene.started` → `scene.drafted` → `scene.critique.report`
(×5) → `scene.editing` → `scene.humanising` → `scene.committed` →
`chapter.committed` → (end) `story.batch_done` OR `story.completed` OR
`story.failed`.

## 6. Read the story

### `GET /api/v2/stories/<story_id>/`

Reader-facing detail — ONLY characters + chapters + conclusion.
Everything else (arc plan, world bible, beat sheets, mood vectors,
critic findings) is hidden behind `/skeleton/`.

**Body:** none

**Response 200:**
```json
{
  "story_id": "9f7e...",
  "title": "The Long Winter",
  "status": "completed",
  "progress": { "stage": "completed", "percent": 100 },
  "created_at": "2026-05-11T12:34:00Z",
  "updated_at": "2026-05-11T12:48:00Z",
  "hidden": false,
  "characters": [
    {
      "character_id": "c1",
      "name": "Mara Holt",
      "role": "protagonist",
      "tier": "main",
      "short_description": "the town's only doctor, exhausted and stubborn",
      "portrait_blurb": "Forty, lean, smells of camphor."
    }
  ],
  "chapters": [
    {
      "chapter_number": 1,
      "title": "The Long Walk Home",
      "summary": "Mara comes back to town in the first storm.",
      "text": "Mara hung her coat by the door. It dripped...",
      "word_count": 2104,
      "committed_at": "2026-05-11T12:40:00Z"
    }
  ],
  "conclusion": null
}
```

`characters` only includes `main` + `recurring` tier — side characters
are hidden from the reader view (visit `/skeleton/characters/` for the
full cast). `conclusion` populates once `status == "completed"`.

## 7. (Optional) Continue, regenerate, edit

### `POST /api/v2/stories/<story_id>/continue/`

Resumes after a batch. Status must be `awaiting_continue`.

**Body (optional):** `{ "additional_chapters": 3 }`. Defaults to 3 or
the remaining chapter count, whichever is smaller.

### `POST /api/v2/stories/<story_id>/regenerate-scene/`

Surgical re-do of ONE scene with optional user notes.

**Body:**
```json
{ "chapter_idx": 2, "scene_idx": 1, "user_notes": "Make Mara colder. She suspects Sam is lying." }
```

**Response 200:** the updated scene draft summary.

### `PATCH /api/v2/stories/<story_id>/scenes/<chapter_idx>/<scene_idx>/`

User edits prose by hand. Optional auto-rerun of the humaniser pass
(strips any em-dashes the user pasted in).

**Body:**
```json
{ "prose": "Mara hung her coat by the door. It dripped...", "run_humaniser": true }
```

### `POST /api/v2/stories/<story_id>/regenerate-from/`

Cascade redo: wipe everything from `(chapter_idx, scene_idx)` forward,
preserve the arc plan, re-launch the orchestrator. Trims character
mood histories + plot seed events past the cutoff.

**Body:**
```json
{ "chapter_idx": 4, "scene_idx": 0 }
```

**Response 202:**
```json
{
  "story_id": "9f7e...",
  "rewind_to": { "chapter_idx": 4, "scene_idx": 0 },
  "remaining_chapters": 16,
  "status": "queued"
}
```

### `POST /api/v2/stories/<story_id>/hide/` and `unhide/`

Soft-delete toggle. Hidden stories drop out of `GET /stories/`.

---

## 8. (Optional) Inspect engine internals

### `GET /api/v2/stories/<story_id>/skeleton/`

Everything the reader-facing detail hides: arc plan, full character
bibles (including side tier + voice fingerprints + mood histories),
world bible, all beat sheets, all scene drafts (with critic findings +
humanisation reports), continuity ledger, both surveys, character
budget, recent progress log.

Sub-resources for lazy-loading large sections:

- `GET /skeleton/arc/` — arc plan only
- `GET /skeleton/world/` — world bible only
- `GET /skeleton/characters/` — all tiers, full bibles
- `GET /skeleton/chapters/<n>/beats/` — one chapter's beat sheet
- `GET /skeleton/chapters/<n>/scenes/` — one chapter's scene drafts + critic findings
- `GET /skeleton/events/?after_seq=N` — run-event log replay

---

## Minimal happy-path summary

```
[optional] POST /api/v2/profiles/generate/        → profile_id
           POST /api/v2/stories/quick/             → story_id  (+ envelope, world, cast persisted)
[optional] PATCH /api/v2/bibles/<id>/character/... → user edits cast
           POST /api/v2/stories/<id>/start/        → 202 queued
           GET  /api/v2/stories/<id>/stream/       → SSE live events
           GET  /api/v2/stories/<id>/              → final story (characters + chapters + conclusion)
[optional] POST /api/v2/stories/<id>/continue/     → next batch
```
